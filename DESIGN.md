# 버팀목 디자인 노트

`getdesign.md`로 Notion, WIRED, Spotify의 인터페이스를 참고해 버팀목에 맞게 정리한 기준입니다.

## 방향

- 화면은 카드 묶음이 아니라 한 권의 문장집처럼 보이게 한다.
- 문장 리더는 종이 면, 얇은 선, 넓은 여백, 절제된 타이포그래피를 기준으로 한다.
- 조작부는 설명 문구보다 익숙한 아이콘과 위치로 이해되게 만든다.
- 음악 플레이어만 어두운 미디어 바처럼 분리하고, 본문 화면은 밝고 조용하게 둔다.
- AI 생성물처럼 보이는 과한 그라디언트, 장식 배경, 불필요한 안내 문장은 쓰지 않는다.

## 색

- Canvas: `#fffdf7`
- Paper: `#ffffff`
- Ink: `#111111`
- Muted: `#6f6a61`
- Hairline: `#ded9cf`
- Soft surface: `#f4f0e8`
- Audio surface: `#151515`
- Audio accent: `#1ed760`

## 타이포그래피

- 모든 화면은 자체 호스팅 Pretendard를 사용한다.
- 제목은 작고 단단하게, 본문은 화면 크기에 맞춰 줄어들되 스크롤이 생기지 않게 한다.
- 모바일과 데스크톱 모두 같은 폰트 계열을 사용해 환경별 인상 차이를 줄인다.

## 컴포넌트

- 상단은 브랜드와 `둘러보기 / 저장함`을 감싸는 하나의 조용한 내비게이션만 둔다.
- 문장 리더는 제목, 메타, 저장 버튼, 본문을 하나의 읽기 면 안에 배치한다.
- 데스크톱에서는 좌우 화살표와 하트가 읽기 면 안에 들어간다.
- 모바일에서는 좌우 화살표와 하트를 하단에 두고, 하트가 가운데에 오도록 한다.
- 저장함 빈 상태는 `저장함`과 `0개`만 보여 중복 안내 문장을 만들지 않는다.

## 금지

- 전체 다크 테마 금지. 음악 바에만 어두운 색을 사용한다.
- 카드 안의 카드, 과한 장식 테두리, pill 버튼 남발 금지.
- 사용자가 이미 알 수 있는 문구, 예를 들면 “눌러서 듣기” 같은 설명은 쓰지 않는다.
